    What is Stone.js?

    Stone.js is a future-ready framework designed for full-context development:
    cloud, serverless, browser, and beyond.

    Continuum Architecture

    One mindset across backend, frontend, serverless, and browser. Learn once, build anything.

    Universal by Design

    Designed for the next generation of web development. Run on Node.js, AWS Lambda, Browser, or anywhere, adapt to any platform.

    Modular & Compact

    No node_modules. One artifact. Pure ESM. Built for performance and portability. Ship your app as a single file, with zero install-time dependencies.

    Declarative & Functional API

    Start Small
    Scale Everywhere

    From microservices to monoliths, from local to cloud, Stone.js lets you scaffold, shape, and ship your app in minutes. Choose your paradigm, pick your platform, and let Stone.js handle the rest. 

    
import { NodeHttp } from "@stone-js/node-http-adapter"
import { IncomingEvent, IEventHandler, StoneApp } from "@stone-js/core"

@NodeHttp()
@StoneApp()
export class App implements IEventHandler<IncomingEvent> {
  /**
   * Handle incoming events
   * 
   * @param event Incoming event
   * @returns response
  */
  handle(event: IncomingEvent): { message: string } {
    const message = `Hello ${event.get<string>('name', 'World')}!`

    // Return a JSON response
    return { message }
  }
}

    
import { defineStoneApp, IncomingEvent } from "@stone-js/core"
import { nodeHttpAdapterBlueprint } from "@stone-js/node-http-adapter"

/**
 * Handle incoming events
 * 
 * @param event Incoming event
 * @returns response
*/
export const Handler = (event: IncomingEvent): { message: string } => {
  const message = `Hello ${event.get<string>('name', 'World')}!`

  // Return a JSON response
  return { message }
}

export const App = defineStoneApp(
  Handler, {}, [nodeHttpAdapterBlueprint]
)

    • 1. Scaffold in seconds

      Run npm create stone and pick your flavor: mono-routed or multi-routed, backend or frontend, React or Vue(soon). No templates, just choices.

    • 2. Choose your paradigm

      Prefer functions? Use the imperative API. Love decorators? Go declarative. Stone.js supports both, equally powerful, equally simple.

    • 3. Run locally. Deploy globally.

      Whether it’s Node.js, Lambda, or the browser, your app becomes one artifact ready to deploy anywhere. No config hell. Just build and run.

    Continuum in Action

    Stone.js isn't built on layers, it's built on a flow, from raw events to rendered experiences. That flow is called the Continuum.

    Setup

    Produce your app’s blueprint by configuring services, features, and plugins, declaratively or imperatively.

    →

    Integration

    Adapt external stimuli (HTTP, CLI, Lambda, browser) into contextual events Stone.js understands.

    →

    Initialization

    Populate the app context, register lifecycle hooks, and prepare for execution.

    →

    Functional

    Handle intentions using event handlers to produce responses, domain logic, clean and portable.
